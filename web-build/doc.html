目錄說明

src

  • 模組的source code

    • dist

    build-scripts
    • 存放build完成的js檔
    • 不會上git
    assets
    • webpack-dev-server測試的assets路徑
    • npm start啟動dev-server (編寫模組測試用)
    index.html
    • dev-server的index page

    main

  • dev-server 各模組的entry point

    • chunk

  • 模組轉成CDN Script的入口
  • import 模組的 source code， 並做處理轉成網頁可用的方式(ex: 將模組方法名放在全域)
  • 每個js file代表一個module , 在此目錄新增，web-build可選擇的模組會直接套用選項

    • sass

  • 放模組相關的sass file

    • web-build

  • 提供一個頁面可以客製化要bundle的模組
  • npm run web-build來啟動server

    • build

  • production 的webpack entry file. 集合各個模組，也可以手動改要import的模組。
  • npm run build

    • webpack.config.js

  • dev-server用到的webpack config

    • prod.config.js

  • production build script用到的webpack config
  • web-build 模式是基於這份config作客製化
